## 一、项目概述

**目标**：构建一个单文件 Web 程序，按月录入 5 条产品线的收入与成本，自动核算盈利状况，并以 SVG「主水管流程图」直观呈现成本流向与净利润的形成。

**核心核算逻辑**
- 毛利润 = 营业收入 − 直接成本
- 间接成本按**各产品线直接成本占比**分摊（最初为按收入占比，后改为按直接成本占比）
- 税前净利润 = 毛利润 − 分摊间接成本
- 净利润 = 税前净利润 × (1 − 税率)（亏损月份不计提所得税，保守口径）
- 经营现金流 ≈ 净利润 + 折旧 − 应收账款增加 − 可用库存增加

**5 条产品线**
| 产品线 | 类型 | 直接成本字段 |
|---|---|---|
| DAISO 填料 | 代理 | 直接采购成本、包装运输成本 |
| 自营填料 | 自营 | 生产人力、材料、生产水电煤租赁、设备采购租赁、包装运输 |
| COSMOSIL 柱 | 代理 | 直接采购成本、包装运输成本 |
| 自营色谱柱 | 自营 | 生产人力、材料、生产水电煤租赁、设备采购租赁、包装运输 |
| 其他产品 | 自营 | 生产人力、材料、生产水电煤租赁、设备采购租赁、包装运输 |

## 三、技术方案

| 维度 | 方案 |
|---|---|
| 前端 | 单文件 `index.html`（内嵌 HTML + CSS + JS，无构建步骤） |
| 持久化 | `localStorage`（key `costflow_<月份>` + 索引 `costflow_index`）；支持导出/导入 JSON、月份切换 |
| 可视化 | SVG 动态渲染（`renderFlow()` 生成 `<svg>` 注入 `#flowChart`）；`compute()` 核算引擎 |
| 发布 | 页面填入 GitHub PAT 后点「☁️ 发布到仓库」，经 GitHub Contents API（`PUT /repos/.../contents/data/costflow.json`，Bearer 令牌）一键写入仓库并触发 Pages 重建；令牌仅存本机浏览器 `localStorage`（无需本地服务器/域名） |
| 建站 | GitHub Pages（main 分支根目录）+ `.nojekyll` |
| 凭证 | 复用 fx-tracker 仓库 `.git/config` 中的 PAT，经 `git -c url...insteadOf` 传入，**不写入** 本仓库配置 |
| 自动更新 | GitHub Actions（`.github/workflows/auto-update.yml`，每日北京时间 9:30 在 GitHub 云端运行，cron `30 1 * * *` + 手动触发；参照 exchange-rate 项目模式，不依赖本机） |
| 坐标校验 | Node 沙箱跑真实 `renderFlow` 生成 SVG，Python 解析 rect/polygon 坐标查越界与重叠 |

### 查看方式（无需本地服务器）
本程序为单文件静态页面，**查看不依赖任何本地服务进程**（早期曾用 `server.py` + `launchctl` 常驻提供预览，已废除；参照 exchange-rate 项目的「纯静态 + GitHub Pages」模式简化）。

- **方式 A（推荐，离线）**：直接双击 `index.html`，以 `file://` 打开即可正常录入 / 计算 / 查看流程图；从仓库载入「已发布数据」在 `file://` 下会被浏览器拦截，但代码已 `catch` 静默处理，不影响本地使用。
- **方式 B（换设备 / 对外）**：用线上站 https://caiclhy0806.github.io/cost-flow-analysis/

> 背景：WorkBuddy 对话内启动的进程只活在当前会话，此前「本地预览反复打不开」即源于此（含沙箱无法 `launchctl` 加载 plist）。改用上述方式后该问题彻底消除，`server.py` 与 `launchd plist` 已删除。

### 一键云发布（无需域名 / 服务器）
1. 页面下方「GitHub 令牌(PAT)」框填入你的 PAT（需 `repo` 权限），点「记住」——令牌仅存本机浏览器 `localStorage`，不清空不会被泄露到代码里。
2. 点「☁️ 发布到仓库」：页面直接用 GitHub Contents API 把 `data/costflow.json` 提交到 `caiclhy0806/cost-flow-analysis` 仓库 `main` 分支，Pages 随即重建。
3. 原理：浏览器 `fetch` 调 `PUT /repos/{owner}/{repo}/contents/data/costflow.json`（先 `GET` 取 sha，再带 sha `PUT`），GitHub API 支持 CORS，故 `file://` 与 `github.io` 来源均可直接调用，**完全不需要任何中转服务器或自定义域名**。
